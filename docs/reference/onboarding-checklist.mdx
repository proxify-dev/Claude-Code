---
title: "Onboarding Checklist"
description: "Quick-start checklist for engineers setting up Claude Code — all actions, no explanations."
---

Get from zero to productive in 30 minutes. Each item links to the full guide if you need details.

- [ ] **Install Claude Code** — [Official quickstart](https://code.claude.com/docs/en/quickstart). Pin a stable version if needed.
- [ ] **Set up Git** — Back up important files. Every project should have version control before you start delegating.
- [ ] **Write your first CLAUDE.md** — 15 minutes. Tech stack, build commands, top 3 conventions. [Full guide →](../setup/claude-md/)
- [ ] **Set up terminal aliases** — `c` for claude, `ch` for chrome mode. [Terminal setup →](../setup/terminal-setup/)
- [ ] **Configure status line** — `/statusline` to show context usage and branch. [Terminal setup →](../setup/terminal-setup/)
- [ ] **Build your skills stack** — Browse [skills.sh](https://skills.sh) and `npx skills search <keyword>`. [Skills stack →](../setup/skills-stack/)
- [ ] **Set up hooks** — Auto-lint, block .env commits. [Hooks playbook →](../setup/hooks-playbook/)
- [ ] **Wire up MCP** — GitHub MCP at minimum. [MCP wiring →](../setup/mcp-wiring/)
- [ ] **Try voice input** — run `/voice`, then hold `Space` to dictate. [Voice dictation docs →](https://code.claude.com/docs/en/voice-dictation)
- [ ] **Run your first plan-then-execute cycle** — Enter `/plan`, define the task, review the plan, execute. [Workflow patterns →](../patterns/workflow-patterns/)
- [ ] **Try 2 parallel sessions** — Open two terminal tabs, run Claude in each. [Parallel execution →](../patterns/parallel-execution/)
- [ ] **Read common pitfalls** — [Common pitfalls →](common-pitfalls/)

<Tip>This checklist takes about 30 minutes. After that, start delegating real work — you learn fastest by doing.</Tip>
